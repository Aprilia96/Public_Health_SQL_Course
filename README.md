# SQL & R for UK Public Health — from zero
### Course project directory

An interactive course (105 lessons, 10 modules) teaching SQL and R for data
analysis in UK public health settings — NHS trusts/ICBs, UKHSA, and local
authority teams. Ships as **one self-contained HTML file** with a live,
in-browser SQLite playground of fictional health data.

---

## Quick start

Open **`course.html`** in any modern browser. That single file contains
everything: all lessons, quizzes, exercises, and the SQL playground.
No server, no install.

---

## Folder map

| Path | What it is |
|---|---|
| `course.html` | The assembled, ready-to-use app — the file you open or share |
| `src/part1_shell.html` | Document head, all CSS, and the page skeleton |
| `src/content.txt` | **All course content** — 105 lessons in a simple markup (edit here) |
| `src/part3_engine.html` | The JavaScript engine: markup parser, renderers, router, progress tracking, and the SQL playground + deterministic data generator |
| `images/` | Diagram PNG files (generated by `tools/gen_images.py`, embedded in the PDF book) |
| `tools/build.sh` | Rebuilds `course.html` from the three `src/` parts, then validates |
| `tools/validate.js` | Structural validator (lesson counts, quiz shape, block types…) |
| `tools/harness.js` | Integration test: boots the real engine in Node, renders every lesson, builds and queries the playground database |
| `tools/gen_images.py` | Generates the diagram PNG files from the course content |
| `tools/build_pdf.py` | Generates the 201-page PDF book edition from `src/content.txt` + embedded images |

The app is a single-page application: the three `src/` parts concatenate,
in order, into `course.html`. There are no other pages.

## Rebuilding after an edit

Linux / macOS:

    bash tools/build.sh

Windows (PowerShell or cmd):

    copy /b src\part1_shell.html + src\content.txt + src\part3_engine.html course.html
    node tools\validate.js

The validator must report `ALL CHECKS PASSED` before you ship a rebuild.

Optional deeper test (requires Node and one dependency):

    npm install sql.js
    node tools/harness.js

Optional PDF book (requires Python 3 with `reportlab`, and the DejaVu fonts):

    python3 tools/build_pdf.py        # writes course_book.pdf here

## Editing or adding lessons — markup reference

Everything lives in `src/content.txt`.

**File header** — `#MODULES` followed by one `number|Title` line per module.

**Lessons** — each starts with `#LESSON <moduleNumber>|<Lesson title>`.
Lesson numbers (1–105) come from file order, so inserting a lesson renumbers
everything after it.

**Blocks** — every block starts on a line beginning `::`

| Block | Syntax |
|---|---|
| Prose | `::p` — blank-line-separated paragraphs; lines starting `- ` become bullets; inline `**bold**`, `` `code` ``, `[text](https://url)` |
| Subheading | `::h\|Heading text` |
| Code | `::code\|lang\|Optional label` — body until next block. `lang` of `sqltry` adds a **Run ▶** button that opens the query in the playground |
| OS-specific code | `::oscode\|lang` with `@win`, `@mac`, `@linux` section markers → rendered as tabs, auto-selecting the reader's OS |
| Quiz | `::quiz` — a `Q ` question line, `- ` wrong options, exactly one `* ` correct option, and a `! ` explanation |
| Exercise | `::ex` — task text, then one or more fenced solution parts each opened by `~~~lang` |
| Key points | `::key` — `- ` bullet lines |
| Callout | `::note\|type\|Optional label` — type is `info`, `warn`, `ai`, or `study` |
| Table | `::table` — pipe-separated rows, first row is the header; use `¦` where a cell needs a literal pipe (e.g. SQL `¦¦`) |

Rules: never start a content line with `::` unless it opens a block, and never
include the character sequences `</script` or `<!--` anywhere in the content
(write `&lt;` / `&gt;` entities if you truly need them — the engine decodes
them on load).

## The playground database

Built deterministically in the browser (seeded PRNG, seed `20260703`) so every
reader gets identical fictional data. Tables:

    patients, gp_practices, lsoa_imd, admissions, icd10_lookup,
    vaccinations, lab_results, deaths, la_population

Roughly 715 patients and 855 admissions across three fictional local
authorities (Barchester, Millford, Kingsbay), tuned so the capstone analyses
have real signal: 30-day emergency readmissions, deaths shortly after
discharge, an MMR-uptake deprivation gradient, frequent attenders, raised
HbA1c/BP results, and still-in-hospital NULL discharges.

The SQL engine (sql.js, ~1 MB) is fetched once from cdnjs on first playground
use; every lesson works fully offline. **Reset database** rebuilds the data
from scratch.
